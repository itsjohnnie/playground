Terminal animation

The art and technique of creating motion graphics using only text characters in command-line interfaces.

demo.sh 

                

            

        

    


    
    

        

            
Since the 1960s, programmers have been creating art and animation using nothing but ASCII characters. From early teleprinter games to modern CLI tools, terminal animation remains a unique intersection of programming, design, and creative constraint.

        

    


    
    

        

            
Core techniques

            
Four fundamental approaches to creating terminal animation.


            

                
                

                    

                        
Frame-based animation

                        
Pre-render multiple frames and cycle through them. The simplest approach, used for logos, loading indicators, and character animations. Memory-intensive but smooth and predictable.

                        

                            const frames = ['⠋', '⠙', '⠹', '⠸']
let i = 0
setInterval(() => {
  process.stdout.write('\r' + frames[i++ % 4])
}, 80)
                        

                    

                    

                        

                            

                        

                    

                


                
                

                    

                        
Procedural generation

                        
Calculate each frame mathematically using functions like sine waves, noise, or physics simulations. Enables effects like plasma, fire, and particle systems with minimal memory.

                        

                            for (let y = 0; y < height; y++) {
  for (let x = 0; x < width; x++) {
    const v = Math.sin(x * 0.2 + time)
    const char = v > 0 ? '█' : '░'
    output += char
  }
}
                        

                    

                    

                        

                            

                        

                    

                


                
                

                    

                        
Buffer manipulation

                        
Maintain a 2D array representing the screen, update specific cells, then render the entire buffer. Allows for complex multi-element scenes like games and interactive UIs.

                        

                            const buffer = Array(rows).fill(null)
  .map(() => Array(cols).fill(' '))

// Update specific cells
buffer[y][x] = '█'

// Render entire buffer
console.log(buffer.map(row => row.join('')).join('\n'))
                        

                    

                    

                        

                            

                        

                    

                


                
                

                    

                        
Character replacement

                        
Use ANSI escape codes to reposition the cursor and overwrite specific characters without redrawing the entire screen. Essential for efficient updates in large displays.

                        

                            // Move cursor to row 5, column 10
process.stdout.write('\x1b[5;10H')

// Write character at that position
process.stdout.write('█')

// Move cursor up 2 lines
process.stdout.write('\x1b[2A')

Character sets

            
Understanding which characters to use for different effects.


            

                

                    
Block elements

                    
█ ▓ ▒ ░ ▄ ▀ ▌ ▐

                    
For solid shapes, gradients, and pixel-style graphics. Unicode U+2580–U+259F.

                


                

                    
Box drawing

                    
─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼

                    
For frames, borders, and diagrams. Unicode U+2500–U+257F.

                


                

                    
Braille patterns

                    
⠀ ⠁ ⠂ ⠃ ⠄ ⠅ ⠆ ⠇ ⠈ ⠉

                    
For smooth curves and high-resolution graphics. Unicode U+2800–U+28FF.

                


                

                    
Geometric shapes

                    
■ □ ▪ ▫ ● ○ ◆ ◇ ★ ☆

                    
For icons, bullets, and decorative elements. Various Unicode ranges.

                

            

        

    


    
    

        

            
Historical context


            

                

                    1963
                    
ASCII standard published, establishing the 7-bit character encoding that would become the foundation for all terminal text.

                


                

                    1978
                    
DEC VT100 terminal released, introducing ANSI escape sequences for cursor control and making sophisticated terminal animation possible.

                


                

                    1985
                    
Bulletin board systems (BBS) popularize ANSI art, creating a thriving community of ASCII artists using tools like TheDraw.

                


                

                    1997
                    
AAlib released, enabling real-time conversion of graphics and video to ASCII art, powering projects like ASCII Quake.

                


                

                    2010s
                    
Modern CLI renaissance brings terminal animation back to prominence through tools like npm progress bars, Docker, and asciinema.

                

            

        

    


    
    

        

            
Notable examples

            
Influential works that pushed the boundaries of terminal animation.


            

                

                    
Star Wars ASCIImation

                    
Simon Jansen, 1997

                    
The entire Star Wars Episode IV recreated in ASCII art, playable via telnet. Originally 16,000+ frames of hand-crafted animation.

                    telnet towel.blinkenlights.nl
                


                

                    
ASCII Quake

                    
AAlib project, 1999

                    
Real-time 3D game rendering in ASCII using dithering algorithms. Demonstrated that even complex graphics could be represented in text.

                


                

                    
Zork

                    
Infocom, 1980

                    
While primarily text-based, Zork's interface animations and status line updates influenced generations of terminal UI design.

                


                

                    
sl (Steam Locomotive)

                    
Toyoda Masashi, 1993

                    
A playful ASCII animation that runs when you mistype 'ls'. Became a beloved Unix easter egg demonstrating smooth terminal animation.

                    brew install sl && sl
